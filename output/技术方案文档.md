# 基于MinerU的医疗文献高质量知识库（RAG）— 技术方案文档

## 1. 整体设计方案

### 1.1 方案概述

本方案针对医疗文献RAG知识库构建的三大核心痛点（复杂版式解析困难、语义级切片准确性低、人工处理效率极低），提出**混合GraphRAG + 医学语义引擎**的创新技术路线。

系统以RAG-Anything框架为底座，叠加四大医学领域创新模块，实现从原始PDF输入到高质量RAG知识库输出的端到端自动化处理。

### 1.3 模型资源
- 文本模型：DeepSeek-V4-Pro（OpenAI兼容接口）
- 视觉模型：Kimi-K2.6（多模态VLM）
- Agent模型：GLM-5.1（降级后备）
- Embedding：text-embedding-3-large (3072维)
- 文档解析：MinerU 3.1（GPU加速pipeline）

## 2. 关键技术的实现路径

### 2.1 复杂版面精准识别与语义级解析

**痛点直达**：医疗文献双栏/多栏复杂版式、基线表/结局指标表、交叉引用内容

**实现路径**：
1. **MinerU精准配置**：lang=ch, backend=pipeline, formula=True, table=True，针对中文医学文献优化
2. **预处理增强**：自动检测水印遮掩区域并增强图像，检测横板表格并旋转校正
3. **医学VLM兜底**：对低质量扫描件使用Kimi-K2.6 VLM直接描述，避免信息丢失
4. **解析缓存**：基于文件hash的解析结果缓存，避免重复GPU计算

**验证指标**：
- 正文提取完整率 > 95%
- 表格结构保留率 > 90%
- 图注-正文匹配准确率 > 85%

### 2.2 基于语义的智能文本切分

**痛点直达**：固定字数拆分 → 语义碎片化 → 大模型幻觉核心诱因

**实现路径**：
1. **PICO框架识别**：自动提取Population / Intervention / Comparison / Outcome
2. **临床试验章节分类**：Primary Outcome / Secondary Outcome / Subgroup Analysis / Sensitivity Analysis / Safety
3. **医学实体标注**：疾病、药物、统计量（RR/OR/HR、95%CI、p值）自动识别
4. **LLM驱动结构解析**：DeepSeek-V4-Pro识别文献逻辑结构，按语义边界切分

**创新点**：拒绝"一刀切"的字符数切分，改为"一刀切在逻辑边界上"。每个chunk既是一个检索单元，也是一个完整的医学论证单元。

### 2.3 端到端知识库自动化构建

**实现路径**：
1. RAG-Anything端到端pipeline：解析→分块→注入→检索
2. 批量并行处理：batch_parser + async并发
3. Gradio Web界面：一键上传、批量导入、实时问答
4. FastAPI接口（可扩展）

## 3. 系统性能与工程化说明

### 3.1 处理效率
- 单篇文献处理时间：< 5分钟（GPU加速）
- 知识库查询响应：< 3秒（双路检索）
- 并行处理能力：支持4篇文献并发解析

### 3.2 容错设计
- **API重试**：3次指数退避（1s→2s→4s）
- **模型降级链**：DeepSeek-V4-Pro → GLM-5.1 → 缓存兜底
- **VLM降级**：Kimi-K2.6 → 纯文本LLM图表描述
- **解析失败兜底**：自动标记 + VLM文本描述

### 3.3 检索准确度提升
相比传统RAG方案：
- **语义完整性**：PICO框架切分，消除碎片化 → 关联检索准确率预计提升30-50%
- **多模态覆盖**：VLM理解图表 → 表格/图像检索覆盖率提升至100%
- **溯源能力**：每条答案附带文献来源+页码+证据等级

## 4. 创新或突破点

1. **PICO+临床试验语义分块**：全球首个将循证医学框架融入RAG切分策略的方案，确保每个chunk是完整的医学论证单元
2. **医学VLM多模态理解层**：不仅仅是OCR图像文字，而是理解森林图的方向性、Kaplan-Meier曲线的临床意义
3. **证据链网络**：跨文献构建证据等级排序和一致性判断，使RAG具备"循证医学思维"
4. **三模型协同架构**：DeepSeek-Kimi-GLM三模型互补，实现TCO与质量的帕累托最优
