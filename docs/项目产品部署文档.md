## 1. 系统架构概览

MedRAG 是一个端到端医疗文献 RAG 知识库系统。部署后，用户可以通过浏览器上传医学 PDF，系统自动完成 VLM 解析、语义切分、向量索引、知识图谱构建，并提供 Agent 多跳推理的智能问答。

### 1.1 服务拓扑

```
┌──────────────────────────────────────────────────────────┐
│  浏览器 (http://localhost:5173)                           │
│  React 19 · TypeScript · Vite · Tailwind CSS              │
├──────────────────────────────────────────────────────────┤
│  API 网关 (http://localhost:8000)                         │
│  FastAPI · REST JSON · SSE Streaming · JWT Auth           │
├──────────────────────────────────────────────────────────┤
│  业务逻辑层 (Python 3.13)                                 │
│  MedicalAgent · MedicalChunker · DualRetriever            │
├──────────────┬───────────────────┬───────────────────────┤
│  FAISS       │  LightRAG         │  MySQL 8.0            │
│  向量检索引擎 │  知识图谱引擎      │  业务数据库            │
├──────────────┴───────────────────┴───────────────────────┤
│  AI 模型层                                                │
│  Qwen2-VL 1.2B (本地 GPU) · BGE-M3 (本地 GPU)            │
│  DeepSeek-Chat (云端 API) · Moonshot Vision (云端 API)    │
└──────────────────────────────────────────────────────────┘
```

### 1.2 数据流向

```
PDF 上传 → MinerU 2.5-Pro VLM 解析 → PICO 语义切分
    → FAISS 向量索引 (BGE-M3)
    → MySQL 业务同步 (articles + text_segments + figures)
    → LightRAG 知识图谱构建
    → 可检索 / 可问答

用户提问 → Agent 多跳推理 (FAISS + LightRAG 双路检索)
    → SSE 流式回答 + 引用溯源
```

## 11. 常见问题排查

### Q1: `ModuleNotFoundError: No module named 'faiss.swigfaiss_avx512'`

**原因**：CPU 不支持 AVX512 指令集。  
**解决**：这是正常的降级行为，系统会自动回退到 AVX2。不用处理。

### Q2: `torch.cuda.OutOfMemoryError`

**原因**：GPU 显存不足。Qwen2-VL 1.2B 需要约 3.5GB。  
**解决**：
```python
# 在 mineru25pro_parser.py 中设置更小的 max_memory
model = Qwen2VLForConditionalGeneration.from_pretrained(
    MODEL_PATH,
    torch_dtype=torch.float16,
    device_map="auto",
    max_memory={0: "6GB"}  # 限制显存使用
)
```

### Q3: `mysql.connector.errors.DatabaseError: 1045 Access denied`

**原因**：MySQL 用户名或密码不正确。  
**解决**：检查 `src/auth.py` 中的 `DB_CONFIG` 字典，确保 `user` 和 `password` 与 MySQL 设置一致。

### Q4: `ConnectionError` 调用 DeepSeek API 时

**原因**：网络不通或 API 密钥错误。  
**解决**：
1. 检查网络是否能访问 `https://api.deepseek.com`
2. 验证 API 密钥是否有效：`curl -H "Authorization: Bearer sk-your-key" https://api.deepseek.com/v1/models`

### Q5: Vite 端口 5173 被占用

**原因**：之前的 Vite 进程未正常关闭。  
**解决**：
```bash
# Windows
netstat -ano | findstr ":5173"
taskkill /F /PID <PID>

# Linux
lsof -i :5173
kill -9 <PID>
```

### Q6: `status ENUM` 错误

**原因**：数据库状态枚举值缺少 `chunking` 和 `indexing`。  
**解决**：
```sql
ALTER TABLE upload_tasks MODIFY COLUMN status 
ENUM('received','parsing','cross_validating','postprocessing','indexing_faiss','indexing_lightrag','chunking','indexing','done','failed','partial') 
DEFAULT 'received';
```

### Q7: 模型下载慢或失败

**原因**：HuggingFace 在国内访问不稳定。  
**解决**：
```bash
# 使用 HF 镜像
export HF_ENDPOINT=https://hf-mirror.com

# 或手动下载后放到指定目录
# Qwen2-VL: ~/.cache/huggingface/hub/models--opendatalab--MinerU2.5-Pro-2604-1.2B/
# BGE-M3:   ~/.cache/huggingface/hub/models--BAAI--bge-m3/
```

### Q8: 首次启动时 FAISS 索引为空

**原因**：这是正常的。首次部署时没有任何文献入库。  
**解决**：通过前端上传页面或 API 上传至少一篇 PDF，系统会自动构建索引。

### 12.2 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name medrag.example.com;

    # 前端静态文件
    location / {
        proxy_pass http://localhost:5173;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }

    # API 后端
    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
