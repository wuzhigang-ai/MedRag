## 一、项目核心定位（必须传达）

MedRAG 是一套**端到端医疗文献 RAG 知识库自动化构建系统**。它将原始 PDF 格式的医学文献（临床指南、Meta 分析、RCT 论文、系统综述等）通过 AI 全自动处理为高质量、可溯源的专业医疗知识库，支撑临床科研、医疗大模型研发与临床决策支持。

**一句话定位**：从 PDF 到智能问答，全链路 AI 驱动的医疗知识引擎。

**核心价值主张**：
- 复杂版式精准解析（双栏/多栏/图表混排 → 结构化知识）
- 语义级智能切分（告别固定字数截断 → PICO 医学逻辑单元）
- 端到端自动化（上传 PDF → 3 分钟后即可智能问答）
- 可溯源专业回答（每条结论精确追溯到文献页码）

### Part 1: 行业痛点与项目背景（3 页）

**第 3 页 — 医疗知识数字化的三大核心痛点**
- 痛点 1：**复杂版式解析困难** — 双栏/多栏混排、医学表格/图表结构丢失，传统工具版式还原准确率极低
- 痛点 2：**语义切分碎片化** — 固定字数拆分破坏章节逻辑，是医疗大模型幻觉的核心诱因
- 痛点 3：**人工处理效率极低** — 单篇文献标准化处理需数小时，无法规模化构建知识库

**第 4 页 — 业务场景**
- 临床科研人员获取文献 → 手动解析/切分/校对 → 构建 RAG 知识库的繁琐现状
- 展示传统流程的痛点：时间成本、错误率、不可规模化

**第 5 页 — 项目目标**
- 实现从原始医疗 PDF 输入到高质量 RAG 知识库输出的**端到端自动化**
- 突破传统固定字数切分，实现基于**医学逻辑结构**的智能文本切分
- 提供**可溯源**的专业医疗问答，每条结论可精确追溯到文献页码

## 三、视觉设计方向指导

### 配色方案
- **主色调**：深蓝渐变（科技感 + 医疗专业性）
  - Primary: #2563EB（蓝）
  - Accent 1: #00C4B4（青 — 医疗感）
  - Accent 2: #D4A853（金 — 品质感）
- **辅助色**：医疗白 (#F8FAFC)、深色文字 (#1E293B)、浅灰背景 (#F1F5F9)
- **图表色板**：
  - FAISS 相关：蓝色系
  - LightRAG 相关：青色系
  - Agent 相关：金色系
  - 医疗数据：绿色系 (#10B981)

### 排版风格
- **字体**：标题用粗体无衬线（如思源黑体 Bold），正文用常规无衬线
- **图表**：优先使用流程图、架构图、对比表、雷达图
- **不要**：密集文字墙、过小字号（正文 < 16pt）、低对比度配色
- **要**：大量留白、图标辅助、数据可视化、层次分明

### 图标与插图方向
- 使用扁平化医学图标（心脏、DNA、文献、图表、神经网络）
- 架构图用干净的分层色块 + 箭头连接
- 对比表用两面卡片式排版（左侧传统方案灰色调，右侧本系统亮色调）
