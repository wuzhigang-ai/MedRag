赛题背景

三大核心痛点

📄

复杂版式解析困难

双栏/多栏、图文混排的医疗文献，现有工具章节切分混乱、表格结构丢失、图注与正文错位，结构化还原准确率低

✂️

固定字数切分失效

传统按字数拆分破坏章节逻辑和语义完整性，Primary/Secondary outcome 等关键信息被截断，是医疗大模型幻觉的核心诱因

⏳

人工处理效率极低

单篇高质量文献标准化处理需数小时，缺乏工程化系统性建设，无法支撑医疗知识库规模化构建

关键技术

四大核心技术突破

① VLM 图表语义理解

Moonshot 128K Vision 自动识别基线表、森林图、KM曲线类型，提取效应量(HR/OR/RR+95%CI)和生存率，结构化注入知识库。超越纯文本RAG的差异化能力。

② 医学语义智能分块

基于章节逻辑(Primary/Secondary outcome)和PICO框架的自适应切分，每个文本块语义完整、可独立检索、来源可追溯。

③ 双引擎混合检索

FAISS 向量检索 + LightRAG 知识图谱(863实体/771关系)互补覆盖。GGraphRAG 捕获实体关系，FAISS 提供精确语义匹配。

④ Agentic 多步推理

6工具 OpenAI Function Calling Agent。检索→交叉验证→证据综合，15步推理上限，SSE流式实时展示工具调用过程。
